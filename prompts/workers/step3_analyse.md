## 4. 分类合理性自检

**必须区分两类情况，处理方式完全不同：**

### A类：文件放错了模块（才能标记 `[分类问题]`）

文件属于另一个现存安全模块，被错误归入了本模块。

**触发条件（必须同时满足全部）：**
- 错放文件占本模块文件总数 **> 30%**
- 目标模块在 `modules/` 中明确**已存在**（先运行 `ls modules/` 确认）
- 是“明显错放”而非“可能更适合”（存在明确的功能边界冲突）

标记格式：
```
[分类问题] <文件路径> 应该归入 <已存在的模块名> 模块，原因：...
```

### B类：文件安全价値低/无关（**不得**标记 `[分类问题]`）

文件是构建脚本、测试用例、文档、打包配置等，并没有应归入的“正确安全模块”。

**对这类文件：**
- **直接分析**，标记为低风险/无威胁，不触发重分类
- 示例：`[B类] 该模块包含 X 个构建/测试文件，无安全威脆，评分：信息级`
- 这类文件应在“模块功能概述”中注明，并将整个模块风险评分降低

**不得标记 `[B类]` 为 `[分类问题]` 将其转化为重分类要求。**

如果分类合理，写 `[分类合理] 所有文件均属于本模块`。

## 5. 威胁分析 (STRIDE)
