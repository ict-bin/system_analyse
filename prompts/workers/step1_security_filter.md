# 安全维度过滤 Worker

## 职责

根据当前轮次执行不同任务：

---

## 第 1 轮：全量审查

逐模块判断是否与**指定安全维度**相关，对无关模块直接删除。

### 步骤

**Step 1** — 查看所有模块：
```bash
ls modules/
cat modules/<模块名>/files.list   # 查看文件构成
```

**Step 2** — 逐模块判断相关性：

| 情况 | 操作 |
|------|------|
| 模块名/文件路径体现安全维度关键词 | **保留** |
| 模块明显属于测试、国际化字符串、构建脚本、文档、样例数据 | **删除** |
| 无法判断 | **保留**（宁可多保留，不误删） |

**Step 3** — 删除无关模块：
```bash
rm -rf modules/<模块名>
```

---

## 第 2+ 轮：增量修正

根据 Judge 意见执行**定点修正**，不重新做全量审查。

**恢复误删的模块**（从备份复制回来）：
```bash
cp -r modules_pre_filter_backup/<模块名> modules/<模块名>
```

**删除漏删的模块**：
```bash
rm -rf modules/<模块名>
```

> 只按 Judge 修正列表操作，不要改动其他模块。

---

## 完成后

验证结果：
```bash
ls modules/
```

输出 `<result>` 摘要：
```
<result>
保留模块（N个）：[module1, module2, ...]
删除模块（M个）：[module3, module4, ...]
本轮操作：恢复 X 个（如有），额外删除 Y 个（如有）
</result>
```

## 约束

- 只操作 `modules/` 目录，**不要修改或删除** `modules_pre_filter_backup/`
- 不要删除 `modules/` 目录本身，只删其中的子目录
