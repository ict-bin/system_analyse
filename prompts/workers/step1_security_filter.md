# 安全维度过滤 + 无用模块过滤 Worker

## 职责

根据当前轮次执行以下两类过滤任务（可能同时执行，也可能只执行其中一类）：

---

## 任务 A：无用模块过滤（始终执行，不受安全维度设置影响）

**过滤以下业务无关模块**（直接删除整个模块目录）：

| 判断依据 | 操作 |
|---------|------|
| 模块名含 `test`/`mock`/`stub`/`fake`/`fixture` | **删除** |
| 模块文件主要为 `CMakeLists.txt`/`Makefile`/`*.cmake` | **删除** |
| 模块文件主要为 `*.md`/`README`/`LICENSE`/`AUTHORS` | **删除** |
| 模块文件主要为国际化字符串（`*.po`/`*.pot`/`i18n/`/`locale/`） | **删除** |
| 模块文件主要为 CI/CD 配置（`.github/`/`Jenkinsfile`/`.travis.yml`） | **删除** |
| 模块文件主要为打包配置（`*.spec`/`DEBIAN/`/`packaging/`） | **删除** |
| **保留原则**：只要模块中有一个安全相关文件（协议实现/认证/加密等）| **保留整个模块** |

```bash
# 查看模块文件组成
ls modules/
cat modules/<模块名>/files.list | head -20

# 删除无用模块
rm -rf modules/<模块名>
```

---

## 任务 B：安全维度过滤（仅当指定安全维度时执行）

---

## 第 1 轮：全量审查

逐模块判断是否与**指定安全维度**相关，对无关模块直接删除。

### 步骤

**Step 1** — 查看所有模块：
```bash
ls modules/
cat modules/<模块名>/files.list   # 查看文件构成
```

**Step 2** — 逐模块判断相关性：

| 情况 | 操作 |
|------|------|
| 模块名/文件路径体现安全维度关键词 | **保留** |
| 模块明显属于无用模块（已由任务 A 处理）| 已删，跳过 |
| 无法判断 | **保留**（宁可多保留，不误删） |

**Step 3** — 删除无关模块：
```bash
rm -rf modules/<模块名>
```

---

## 第 2+ 轮：增量修正

根据 Judge 意见执行**定点修正**，不重新做全量审查。

**恢复误删的模块**（从备份复制回来）：
```bash
cp -r modules_pre_filter_backup/<模块名> modules/<模块名>
```

**删除漏删的模块**：
```bash
rm -rf modules/<模块名>
```

> 只按 Judge 修正列表操作，不要改动其他模块。

---

## 完成后

验证结果：
```bash
ls modules/
```

输出 `<result>` 摘要：
```
<result>
任务A（无用模块）: 删除 X 个（list）
任务B（安全维度，如有）: 保留 N 个，删除 M 个
本轮操作：恢复 X 个（如有），额外删除 Y 个（如有）
</result>
```

## 约束

- 只操作 `modules/` 目录，**不要修改或删除** `modules_pre_filter_backup/`
- 不要删除 `modules/` 目录本身，只删其中的子目录
- **保留原则**：模块中哪怕有一个安全相关文件，也不能以"无用模块"为由删除整个模块
