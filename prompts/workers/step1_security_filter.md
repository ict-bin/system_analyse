# 安全维度过滤 Worker

## 任务目标

Stage 1 全局分类已完成。你的任务是：**直接删除 `modules/` 目录下与指定安全维度无关的模块目录**，仅保留相关模块。

## 目录说明

- `modules/`：当前模块目录（**本轮已从备份还原**，可直接操作）
- `modules_pre_filter_backup/`：过滤前的完整备份（**只读，不要修改或删除**）

## 工作流程

### Step 1：查看所有模块

```bash
ls modules/
```

如需了解某个模块的文件构成：
```bash
cat modules/<模块名>/files.list
```

### Step 2：逐模块判断相关性

对每个模块判断是否与**指定安全维度**相关：

- 模块名称体现安全维度关键词 → **相关，保留**
- `files.list` 中文件路径/名称与安全维度功能明显关联 → **相关，保留**
- 明显属于测试代码、国际化字符串、构建脚本、文档、样例数据 → **无关，删除**
- 无法判断 → **保留**（宁可多保留，不误删）

### Step 3：删除无关模块

对每个判定为无关的模块，执行：

```bash
rm -rf modules/<模块名>
```

### Step 4：验证结果

```bash
ls modules/   # 确认只剩相关模块
```

### Step 5：输出 `<result>` 摘要

```
<result>
保留模块（N个）：[module1, module2, ...]
删除模块（M个）：[module3, module4, ...]
删除原因：
- module3：属于国际化字符串，与安全无关
- module4：构建工具，不涉及指定维度
</result>
```

## 重要约束

- **只操作 `modules/` 目录**，不要碰 `modules_pre_filter_backup/`
- 删除操作使用 `rm -rf modules/<模块名>`，不要移动到其他位置
- 不要删除 `modules/` 目录本身，只删其中的子目录
