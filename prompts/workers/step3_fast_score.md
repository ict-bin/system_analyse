# 任务（仅评分）

综合上方符号/函数信息、模块名与文件清单，对本模块的**安全威胁严重程度**给出一个 0-100 评分与等级，供后续优先级排序。

评分要点（综合权衡，无需展开）：
- **对外攻击面**：网络监听/IPC/导出接口、危险调用（system/exec/popen/strcpy/memcpy/反序列化/SQL 等）
- **数据敏感性**：认证、密钥、加密、权限、配置、凭据
- **模块性质**：核心服务/协议/解析器（高）↔ 构建/测试/文档/示例（低）

# 约束（严格遵守）

1. **只调用一次 `write`**，把下面格式写入 `modules/<模块名>/module_report.md`，写完立即结束。
2. **禁止** read / bash / grep / 探索等任何其他操作（信息已提供）。
3. **禁止**输出 STRIDE 详述、文件清单、功能概述等长篇内容——**只输出下面的评分块**。
4. 使用相对路径（`modules/...`），不要用绝对路径。

# 输出格式（严格照写，保持简短）

```
<!-- RISK_LEVEL: 高 -->
<!-- RISK_SCORE: 78 -->
# <模块名> 安全威胁评分
- 风险等级：高
- 风险评分：78
- 评分依据：<一句话，≤60 字，说明主要风险来源>
<result>风险评分 78（高）：<一句话依据></result>
```

风险等级↔评分对应：`严重` 85-100 / `高` 70-84 / `中` 40-69 / `低` 10-39 / `信息` 0-9。
等级与评分必须一致。`<!-- RISK_LEVEL -->`、`<!-- RISK_SCORE -->`、`<result>` 三者缺一不可。
