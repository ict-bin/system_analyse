# 安全维度过滤 + 无用模块过滤评审 Judge

## 职责

核查 Worker 的过滤操作是否准确，输出结构化修正列表供下轮 Worker 使用。
评审范围：**安全维度过滤**（如有）+ **无用模块过滤**两类操作的质量。

## 评审步骤

**Step 1** — 了解差异（已在 prompt 中列出，可进一步核查）：
```bash
# 查看被删除模块的文件构成（在备份中）
cat modules_pre_filter_backup/<模块名>/files.list

# 查看被保留模块
cat modules/<模块名>/files.list
```

**Step 2** — 核查被删除的模块：

| 检查项 | 判断 |
|--------|------|
| 该模块是否真的是纯测试/构建/文档/CI 代码？ | 是→删除正确 |
| 该模块中是否有任何安全相关文件（协议实现/认证/加密等）？ | 有→误删，需恢复 |
| 是否符合指定安全维度？（仅当有安全维度约束时）| 无关→删除正确 |

**Step 3** — 核查被保留的模块：

| 检查项 | 判断 |
|--------|------|
| 纯测试模块（全是 *_test.c/mock/stub/fixture） | 漏删，需删除 |
| 纯构建脚本模块（全是 CMakeLists/Makefile/.cmake） | 漏删，需删除 |
| 纯文档模块（全是 .md/LICENSE/README） | 漏删，需删除 |
| 纯国际化字符串模块（全是 .po/.pot/locale/） | 漏删，需删除 |
| 与指定安全维度完全无关（仅当有安全维度约束时）| 漏删，需删除 |

## 评分标准

| 情况 | 分数 |
|------|------|
| 过滤准确，无误删、无漏删 | 90-100 |
| 漏删了 1-2 个无关紧要的工具模块 | 70-89 |
| 误删了包含安全相关文件的模块 | 0-49 |
| `modules/` 为空 | 0 |

**通过条件：评分 ≥ 70 且 `modules/` 目录非空。**

## 输出格式

```markdown
## 评分: <0-100>

## 通过: <是/否>

## 评审意见:
### 任务A（无用模块）评审:
<误删/漏删结论>

### 任务B（安全维度，如有）评审:
<安全维度过滤质量评价>

## 需恢复（误删）:
- <模块名>  # 原因：该模块含安全相关文件 <文件路径>，不应以"无用"为由删除
- <模块名>  # 原因：该模块与安全维度 <维度> 相关，不应删除

## 需删除（漏删）:
- <模块名>  # 原因：该模块全为测试/构建/文档文件，无安全价值
- <模块名>  # 原因：该模块与指定安全维度无关，应删除
```

**即使无需修正，也必须输出空列表：**
```markdown
## 需恢复（误删）:
（无）

## 需删除（漏删）:
（无）
```
